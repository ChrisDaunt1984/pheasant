1 Pheasant

Welcome to Pheasant! In this section, overview and some interesting features of Pheasant are presented.

1.1 Overview

Pheasant is a Markdown converter which is designed to work with MkDocs as a plugin.

Highlights include:

  • Auto generation of outputs for a fenced code block or inline code in Markdown source using Jupyter client. The code language is not restricted to Python.
  • Auto numbering of headers, figures, and tables, and etc. Numbered objects can be linked from other Markdown sources.

1.2 Installation

You can install Pheasant from PyPI.

$ pip install pheasant

If you use Pheasant as a plugin of MkDocs, you also need to install it.

$ pip install mkdocs

Then, in your mkdocs.yml, add lines below to register Pheasant as a MkDocs plugin and its theme based on Read the Docs theme:

theme: rtd-pheasant

plugins:
  - pheasant

1.3 Getting Started

1.3.1 Auto generation of the executed outputs with Jupyter client

A markdown source below:

<div class="pheasant-fenced-code"><div class="input"><pre><code class="python">print(1)</code></pre></div><div class="stdout"><pre><code class="python">1</code></pre></div></div>

is converted into:

<div class="pheasant-fenced-code"><div class="input"><pre><code class="python">print(1)</code></pre></div><div class="stdout"><pre><code class="python">1</code></pre></div></div>

<div class="pheasant-fenced-code"><div class="input"><pre><code class="python">1</code></pre></div><div class="output"><pre><code class="python">1</code></pre></div></div>

after execution of print function via Jupyter client and finally rendered as:

print(1)
1

1.3.2 Inline code embeded in a Markdown source

"Inline code" is a powerful feature of Pheasant. Any python codes surrounded by <span class="pheasant-inline-code"><span class="error">SyntaxError: unexpected EOF while parsing (&lt;ipython-input-7-3b078fb80f7e&gt;, line 1)</span></span> are executed and the result remains there. For example, <span class="pheasant-inline-code"><span class="error">SyntaxError: unexpected EOF while parsing (&lt;ipython-input-8-b2cfd68b5a89&gt;, line 1)</span></span> becomes 15. Variables can be assigned in an inline code like this: `. Then,"I'm SyntaxError: unexpected EOF while parsing (<ipython-input-12-18620fe0638c>, line 1)."` becomes "I'm Pheasant." Note that an inline code without outputs does not shown after execution.

1.3.3 Visualization

Pheasant supports various output formats other than standard stream or plain text output. For example, you can create a PNG image using Matplotlib. First, import Matplotlib plotting library.

```python
import matplotlib.pyplot as plt
```
import matplotlib.pyplot as plt

Then, plot a line.

```python
plt.plot([1, 2])
```
plt.plot([1, 2])
[<matplotlib.lines.Line2D at 0x29e385a4b38>]

1.3.4 Auto numbering of headers, figures, tables, etc.

As you can see, all of headers are numbered in this document. This numbering has done by Pheasant automatically. In addition, Pheasant can count the number of figures, tables, etc. and give the identical number to each object.

You can use a special "header" statement for figure, table, etc. to number them like below:

#Figure This is a cat. [1](index.md#cat)
![jpg](img/cat.jpg)

jpg not supported

Figure 1 This is a cat.

Supported numbered headers are shown in the next table:

Table 1 Supported numbered headers

Type Markdown
Header # (title)
Figure #Figure (title), #Fig (title)
Table #Table (title), #Tab (title)
Code #Code (title)
File #File (title)

In the above Markdown source, <span style="color: red;">Unknown tag: '&lt;tag&gt;'</span> is an ID tag for hyperlink described below. Off course, you can use any code to create a figure:

```python display
plt.plot([3, 2])
```
