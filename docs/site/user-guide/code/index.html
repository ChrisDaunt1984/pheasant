4 Code

Pheasant can include codes from Python moudle or a file system.

4.1 File system

Any text files are read from file system by Pheasant's extended Markdown syntax. For example,

![file](mkdocs.yml)

site_name: Pheasant
site_url: https://pheasant.daizutabi.net/
repo_url: https://github.com/daizutabi/pheasant/

theme:
  name: rtd-pheasant

plugins:
  - pheasant

nav:
  - Home: index.md
  - User Guide:
    - Images: user-guide/images.md
    - Code: user-guide/code.md
  - About:
    - Release Notes: about/release-notes.md
    - License: about/license.md

markdown_extensions:
  - admonition
  - pymdownx.arithmatex
#   - attr_list

The root directory of a relative path is the directory where mkdocs.yml exists.

If the file is too long to display the whole content, you can specify the lines as the same way Python does with the slice notation. Use < and > insted of [ and ] because [ and ] can be a part of a file name.

![file](mkdocs.yml<4:10>)

theme:
  name: rtd-pheasant

plugins:
  - pheasant

When you want to specify a language, write like below after ? character:

![file](example.js<:10>?javascript)

Imported file can be numbered like figures and tables. Use this shorthand notation:

#![file](pheasant.yml)

File 4.1 pheasant.yml

jupyter:
  kernel_name:
    python: python3
  sys_paths:
    - scripts
  import_modules:
    - example
number:
  level: 0
code:
macro:

The above file is the real config file of this Pheasant document.

4.2 Python source

Python module sources that the current Jupyter kernel can find are also inspected. Now Pheasant is configured like the above phesant.yml settings, the kernel's sys.path includes scripts directory and module example has already imported.

import sys
sys.path[:5]

['c:/users/daizu/documents/github/pheasant',
 'scripts',
 '',
 'C:\\Users\\daizu\\Documents\\GitHub\\pheasant\\docs',
 'C:\\Users\\daizu\\Miniconda3\\envs\\daizu\\python37.zip']

example

<module 'example' from 'scripts\\example.py'>

You can write to inspect the whole module content:

#![python](example)

Code 4.1 example

"""An Example Module."""

def add(x, y):
    return x + y


def sub(x, y):
    return x - y

A part of the module can be specified as usual.

#![python](example.sub)

Code 4.2 example.sub

def sub(x, y):
    return x - y

With this functionality, you can guarantee the reproducible relation between your source code and results easily.
